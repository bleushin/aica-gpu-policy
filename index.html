본 지침은 다비오(Dabeeo Inc.)에서 작성하였으며, 인공지능산업융합사업단(AICA)에서 제공하는 GPU 자원의 부정 사용을 방지하고 보안 사고를 사전에 예방하여 전체 구성원이 안정적으로 자원을 활용할 수 있도록 함을 목적으로 합니다.

These guidelines, prepared by Dabeeo Inc., aim to prevent the misuse of GPU resources provided by AICA and ensure stable resource utilization for all members by proactively preventing security incidents.

1. 부정사용 및 관리 위반 유형 1. Types of Misuse and Violations

목적 외 사용Out-of-Purpose Use

  • 이용신청/계획서에 명시된 과제 외 목적 사용
  • 가상화폐 채굴(마이닝) 등 부정사용
  • GPU Burn 등 인위적인 이용률 조작 프로그램 실행
  • Use for projects not specified in the application
  • Cryptocurrency mining and other unauthorized use
  • Running programs to artificially inflate GPU utilization (e.g., GPU Burn)

모니터링 방해Monitoring Interference

  • 모니터링 Agent 임의 삭제
  • 기술지원 및 재설치 요구 불응
  • 모니터링 불가 상태 장기 방치
  • Arbitrary deletion of Monitoring Agent
  • Refusal to reinstall requested by support
  • Leaving resources in a non-monitorable state

2. 주요 발생 사례 및 원인 2. Major Misuse Cases and Causes

구분Category 주요 사례 및 원인Case and Primary Cause
Malware GitHub 등에서 트레이닝용 실행파일 다운로드 시 채굴 기능이 포함된 악성코드 설치 사례 Mining malware infection cases through downloaded training scripts/executables from GitHub.
Security 서버 보안 설정 전체 Open(Any), 접속 IP 차단 정책 미적용, 불필요한 test 계정 운영 Wide-open security groups (Any), missing IP whitelist policies, maintaining unnecessary test accounts.
Port Mgmt 추론 서비스 실증 등을 위한 포트 Open 후 미관리 (서비스 종료 후 미폐쇄) Neglecting open ports after inference service verification tasks; failure to close ports post-verification.
Account Mgmt 계정 공유 이후 관리 미흡으로 인한 채굴 프로그램 설치·운영 사례 발생 Instances of mining program operation due to poor account hygiene following account sharing.

4. 위반 시 조치 및 제재 4. Actions and Sanctions

즉시 조치 (자원 회수)Immediate Action

목적 외 사용 확인 시 사전 통보 없이 즉시 자원 회수. 해당 이용자는 사실확인서(공문) 필수 제출. Immediate resource recovery without prior notice upon confirmation of misuse. Users must submit a fact confirmation letter.

데이터 백업 기한 (10일)Backup Period (10 Days)

회수 통보일로부터 10일 이내 백업 완료 필수. 기한 초과 시 데이터 복구 불가. Backup must be completed within 10 days of notice. Recovery is absolutely impossible afterward.

5. 주요 예방·확인 사항 5. Key Preventive Items

자원 사용량 모니터링Usage Monitoring

  • CPU/GPU 사용률 상시 확인
  • 미사용 기간 중 사용률 급증 여부 점검
  • 콘솔 Monitoring 알림 기능 활용
  • Regularly check CPU/GPU utilization
  • Monitor for spikes during idle periods
  • Utilize console monitoring notifications

보안 규칙 설정Security Rules

  • 접속 IP를 0.0.0.0/0으로 설정 금지
  • 특정 IP/Port 기준으로 최소화 생성
  • 허용 IP에 대한 보안 정책 적용
  • Do NOT set IP to 0.0.0.0/0
  • Limit access to specific IPs/Ports
  • Apply strict policies for allowed IPs

6. 방지 대책 (운영 원칙) 6. Security Principles

01

계정 관리Account Security

계정 공유 금지, 퇴사자 즉시 회수, 최소 권한 원칙 적용 No sharing, revoke accounts immediately, least privilege principle.

02

접속 통제Access Control

서버 접속 IP 제한, 업무 필요 IP만 허용 Restrict server access IP to business-essential addresses only.

03

포트 관리Port Control

불필요 포트 개방 금지, 실증 종료 즉시 차단 No unnecessary ports; close ports immediately after verification.

04

정기 점검Regular Audits

주기적 모니터링 및 보안 설정 현황 점검 Periodic monitoring and security configuration audits.
